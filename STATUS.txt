SMARTROUTE PROJECT STATUS

Overview
- Demo-ready multi-agent travel planner with API-based attractions.
- Pricing is not live: booking cards show "Compare on provider" with external links.
- Photos are Creative Commons placeholders by category; some places show "No photo available".

Implemented
- 7 agents with UI status indicators and activity log.
- FastAPI backend with WebSocket updates.
- Q-learning (epsilon-greedy), MCTS optimization, MDP reward tracing.
- Bayesian preference learning (Beta, Dirichlet) and POMDP belief updates.
- Weather forecasts from Open-Meteo.
- Nearest-neighbor routing for itinerary ordering.

Not Implemented (by design)
- DQN, PPO, Hierarchical RL, Gaussian Processes.
- Live pricing integrations (Amadeus, Skyscanner, Kiwi, etc.).
- Real-time crowd APIs.
- Trained ML weather models (current Naive Bayes uses fixed parameters).

Notes
- Booking flows provide guided steps with provider links for final checkout.
- Use provider links to compare and complete bookings.

Project Status: Active demo build
